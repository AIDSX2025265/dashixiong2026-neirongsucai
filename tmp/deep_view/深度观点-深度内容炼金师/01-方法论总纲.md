# CODE方法论：知识炼金四式

## 总览

```
Capture（捕获）→ Organize（组织）→ Distill（提炼）→ Express（表达）
```

---

## 一、Capture（捕获）- 第一步

### 核心理念
- **不要过滤**：任何想法都值得记录
- **最小阻力**：用什么工具最顺手就用什么
- **随机漫步**：想法是随机的，捕获也应该是随机的

### 实践要点
- 快速记录，不要纠结格式
- 用标签连接想法，而非文件夹
- 定期回顾，每周整理一次

---

## 二、Organize（组织）- 第二步

### PARA分类法

| 类别 | 说明 | 判断标准 |
|------|------|----------|
| **P** Projects | 正在进行的项目 | 有明确截止日期 |
| **A** Areas | 长期责任领域 | 无截止日期，需要持续关注 |
| **R** Resources | 参考资源 | 可能在未来有用的材料 |
| **A** Archives | 归档存储 | 不活跃但可能需要参考 |

### 组织原则
- 一个东西只放一个地方
- 按行动频率分类，不按主题分类
- 定期清理：完成的项目归档

---

## 三、Distill（提炼）- 第三步

### DIKW递进提炼

```
Data（数据）
    ↓
Information（信息）
    ↓
Knowledge（知识）
    ↓
Wisdom（智慧）
    ↓
Expression（表达）
```

### 各层级说明

| 层级 | 说明 | 提炼动作 |
|------|------|----------|
| Data | 原始素材、碎片化记录 | 收集、整理 |
| Information | 结构化的信息 | 关联、分类 |
| Knowledge | 提炼出的规律和模式 | 总结、抽象 |
| Wisdom | 融会贯通的洞察 | 思考、升华 |
| Expression | 可传达的内容 | 表达、输出 |

---

## 四、Express（表达）- 第四步

### 四步版式：起承转合

| 步骤 | 内容要点 |
|------|----------|
| **起** | 引入话题，抛出核心观点/问题 |
| **承** | 展开论述，提供论据/案例 |
| **转** | 深入挖掘，转折递进，提出新视角 |
| **合** | 总结升华，给出行动方向 |

### 四种体裁模板

1. **观点类**：观点 → 论证 → 升华 → 行动呼吁
2. **知识类**：问题 → 方法 → 案例 → 总结
3. **叙事类**：背景 → 冲突 → 转折 → 收获
4. **对话类**：引入 → 问答 → 深入 → 延伸

---

## 五、完整流程

```
【想法出现】→ 快速Capture（不过滤）
    ↓
【定期整理】→ Organize（PARA分类）
    ↓
【深度思考】→ Distill（DIKW提炼）
    ↓
【输出内容】→ Express（四步版式）
```

---

## 六、核心原则

1. **流优于存**：让内容流动起来，而不是堆积
2. **行动导向**：所有组织都服务于行动
3. **递进深化**：从数据到表达，层层递进
4. **活人感**：输出要有人的温度和思考痕迹
