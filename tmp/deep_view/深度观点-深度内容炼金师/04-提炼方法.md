# Distill（提炼）：DIKW递进提炼

## 五个层级

```
Data（数据）
    ↓
Information（信息）
    ↓
Knowledge（知识）
    ↓
Wisdom（智慧）
    ↓
Expression（表达）
```

---

## 层级详解

### Data（数据）- 原始素材

**是什么**：
- 碎片化的记录
- 未加工的信息
- 原始的观察

**例子**：
- 一段聊天记录
- 文章中的一句话
- 突然冒出的想法
- 看到的一个现象

**提炼动作**：收集、记录

---

### Information（信息）- 结构化信息

**是什么**：
- 经过整理的数据
- 有上下文的信息
- 可关联的内容

**例子**：
- 整理后的笔记
- 分类好的素材
- 带标签的记录

**提炼动作**：分类、关联、梳理

---

### Knowledge（知识）- 提炼的规律

**是什么**：
- 从信息中总结的规律
- 可复用的模式
- 抽象出来的方法论

**例子**：
- 标题爆款公式
- 文案框架模板
- 工作流方法

**提炼动作**：总结、抽象、模式化

---

### Wisdom（智慧）- 融会贯通的洞察

**是什么**：
- 深度思考后的洞察
- 跨领域连接的新认知
- 改变行动的观点

**例子**：
- "AI不是代笔者，是对话者"
- "说出来就有了"
- "难而正确，大概说的就是这种东西"

**提炼动作**：思考、连接、升华

---

### Expression（表达）- 可传达的内容

**是什么**：
- 能被他人理解和接受的形式
- 有结构和逻辑的表达
- 带有个人风格的内容

**例子**：
- 一篇完整的文章
- 一次有观点的分享
- 一段有启发的对话

**提炼动作**：表达、输出、传播

---

## 提炼流程

### 从Data到Information
```
碎片记录 → 加上上下文 → 分类整理 → 可检索的信息
```

### 从Information到Knowledge
```
多条信息 → 发现模式 → 总结规律 → 可复用的知识
```

### 从Knowledge到Wisdom
```
既有知识 → 深度思考 → 跨域连接 → 个人洞察
```

### 从Wisdom到Expression
```
核心观点 → 四步版式 → 加入风格 → 完整表达
```

---

## 实践建议

### 1. 不要跳级
- 先有足够的Data，再提炼Information
- 先积累Information，再总结Knowledge
- 先消化Knowledge，再形成Wisdom

### 2. 循环往复
- 每次表达都会产生新的Data
- 表达是检验提炼效果的最佳方式

### 3. 记录过程
- 不仅记录结果，也记录思考过程
- 过程往往是更宝贵的内容

---

## 与内容创作的结合

| DIKW | 内容创作中的对应 |
|------|----------|
| Data | 日常捕获的想法、素材 |
| Information | 整理后的笔记、素材库 |
| Knowledge | 文案框架、方法论 |
| Wisdom | 个人观点、独特洞察 |
| Expression | 最终发布的文章 |
