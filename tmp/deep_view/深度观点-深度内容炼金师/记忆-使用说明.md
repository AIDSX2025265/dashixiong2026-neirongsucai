# 深度观点·记忆

## 基本信息

- **创建日期**：2026-02-12
- **用途**：观点深挖，按CODE炼金四式流程
- **定位**：区别于AI创作工作流，不做长文创作，只做观点深挖

---

## CODE工作流（内部执行）

```
Step 1 - Capture（捕获）：提取核心Point
Step 2 - Organize（组织）：理解观点属性
Step 3 - Distill（提炼）：延伸连接，挖掘新洞察
Step 4 - Express（表达）：输出完整内容
```

**一次性输出，不分段**

---

## 输出格式

```
【内部思考】
- 提取的核心观点
- 延伸的连接方向
- 挖掘的新洞察

【输出内容】
（可直接发布的完整内容）

---

是否认同？是否存档？
```

---

## 语言风格要求

| 要求 | 说明 |
|------|------|
| 去掉口癖 | "说实话"、"我觉得"、"你会发现"等全部删除 |
| 语言精准 | 不用口癖模仿语气，直接精准表达 |
| 去掉废话结尾 | "难而正确"等全部去掉 |
| 保持本质 | 口语化、真实、利他、发心要正 |

---

## 深挖规则（重要）

### 1. 场景优先
- 深挖时优先用场景化例子
- 不要只有抽象阐述
- 例子要让读者"看到"画面

### 2. 观点住进案例
- 观点和案例要融合
- 不要分成"观点+例子"两段
- 观点"住"在例子里，不是"放"在例子前

### 3. 控制字数
- 能说清楚就行
- 不要啰嗦
- 字数根据内容需要调整

---

## 存档规则

- 用户说"存档"后，保存为独立文件
- 文件名格式：日期-核心观点.md
- 存放位置：存档/文件夹
- 包含：原始观点 + 深挖内容
