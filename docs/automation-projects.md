## 2. 小红书 → 提图 → 生图 → 回传

### 项目目标
读取小红书笔记内容，提取图片，用工作流生成新图，再回传给用户。

### 核心链路
1. 读取小红书笔记
2. 提取图片 URL
3. 调用生图工作流
4. 下载到本地
5. 逐张发回用户

### 已确认使用的模块
- `xhs-note-extractor`
- `nanobananapro-image-gen`

### 已确认工作流特点
- 使用 Coze 工作流
- 返回为 SSE 流式响应
- 解析终点节点 `End`
- 通过临时文本文件在模块之间传递图片 URL 列表

### 最重要的执行原则
**必须串行，不要一次性批量生成。**

推荐流程：
1. 读笔记
2. 拿图片列表
3. 一张一张生成
4. 一张一张发送
5. 每一步给进度反馈

### 原因
- 批量生成容易触发速率限制
- 用户会觉得“卡住”或“看不到过程”
- 串行更稳，也更符合大师兄偏好

### 常见文件痕迹
- `watch_images_list.txt`
- `images_list.txt`
- 技能目录中的 Coze 配置样例

### 当前接手时的建议动作
1. 先确认链接提取模块能否工作
2. 再确认 Coze token 是否还有效
3. 生成时默认逐张执行
4. 对用户保持进度可见
