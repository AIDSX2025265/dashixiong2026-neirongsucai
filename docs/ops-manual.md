## 3. 微博 → 飞书 → 公众号：执行手册

### 目标
采集微博内容，整理到飞书，必要时传到公众号草稿箱。

### 可选方案

#### 方案 A：完整自动化
适合：想一条龙跑完
核心：`weibo-to-wechat`

#### 方案 B：扩展界面操作
适合：想可视化勾选
核心：Chrome 扩展版

#### 方案 C：快速上传测试
适合：快速验证公众号上传
核心：`simple_upload.py`

### 默认执行顺序
1. 检查配置是否还有效
2. 检查微博 cookie 是否还活着
3. 跑微博采集
4. 确认本地图片是否真实下载
5. 同步到飞书
6. 如需，再传公众号

### 已知关键风险
- 微博 cookie 可能失效
- 直接抓图可能拿到占位图
- 公众号 API 受 IP 白名单限制
- 飞书写入可能受权限或字段映射影响

### 处理原则
- 微博侧：优先用成熟方案，不重新发明抓取方式
- 飞书侧：偏数据承接和展示
- 公众号侧：执行前先确认白名单

## 4. 小红书 → 生图：执行手册

### 标准流程
1. 读取小红书笔记
2. 提取图片 URL 列表
3. 调用生图工作流
4. 下载结果
5. 一张一张回传

### 必守原则
**不要批量一次性生成。**
默认必须：
- 串行生成
- 串行发送
- 每一步给进度

### 为什么
- 更稳
- 降低速率限制风险
- 用户能看见过程
- 更符合大师兄偏好

### 默认说法 / 默认动作
如果用户只说“读这个小红书，然后生图”：
我默认自动拆成：
- 先读链接
- 再提图
- 再逐张生成
- 再逐张回传

不额外反复确认。
