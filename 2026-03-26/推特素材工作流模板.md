# 推特素材工作流模板

## 一、日常标准流程

### 第一步：先扫素材
你对我说：

`扫今天素材，给我排序`

我会返回给你：

- 今天值得看的素材方向
- 按优先级排序
- 每条一句话说明
- 哪些更适合做完整素材包

### 第二步：你选
你对我说：

`做 1、2、4 素材包`

或者：

- `做前 3 个`
- `只做公众号自动化和 Chrome MCP`
- `这 5 条都做`

### 第三步：我做完整素材包
我会按你确认的内容，输出完整素材包。

每条素材包固定包含：

- 原帖正文
- 原帖图片 / 视频素材
- 图中提取出的落地链接
- 评论区补充素材
- 评论区里的落地链接 / 图片
- 必要的延伸落地链接

### 第四步：入库
你对我说：

`把已确认素材入库`

我就会：

- 按当天日期建文件夹
- 写成 md
- 进入素材仓库 / 知识库

## 二、可直接复制的指令

### 1. 先看今天有什么
`扫今天素材，给我排序`

### 2. 直接做素材包
`做 1、3、5 素材包`

### 3. 全部做完整素材包
`把今天前 5 条都做成素材包`

### 4. 入仓库
`把今天确认过的素材包入库`

### 5. 只找某个方向
- `只找今天 AI workflow 相关素材，给我排序`
- `只找今天 OpenClaw / ClawBot 相关素材，给我排序`
- `只找今天适合公众号写的素材，给我排序`

## 三、完整素材包标准格式

```md
# 素材包 xx

## 原帖正文
……

## 原帖图片 / 视频
- 图片1：
- 图片2：

## 原帖/配图中提取出的落地链接
- 教程：
- GitHub：
- 官网：
- 文档：

## 评论区补充素材
### 补充信息
- ……

### 评论区里的落地链接
- ……

### 评论区里的图片
- ……
```

## 四、默认规则

### 1. 先给素材本体，再谈创作
不先分析，不先写观点。

### 2. 默认不给原帖链接和评论链接
除非后续单独要求。默认只给素材本体、图片、落地链接。

### 3. 有落地链接必须采
比如：

- GitHub
- 教程
- 官网
- 文档
- skill
- 工具地址

### 4. 评论区只保留有用的
只留：

- 补充信息
- 落地链接
- 图片素材

## 五、最推荐的日常用法

每天按这 3 句来：

### 早上
`扫今天素材，给我排序`

### 选完后
`做 1、2、4 素材包`

### 最后
`把已确认素材入库`

这样最省时间，也最顺。
